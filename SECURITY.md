# Security Policy

## Supported Versions

Only the latest public release is supported with security fixes.

Older versions may not receive patches.

## Reporting A Vulnerability

If you discover a security issue, please do not open a public GitHub issue.

Please report it privately to:

- GitHub Security Advisories, if enabled for this repository
- or by contacting the maintainer directly

When possible, include:

- affected version
- clear reproduction steps
- expected vs actual behavior
- potential impact
- logs, screenshots, or proof of concept if relevant

## What To Report

Examples of issues worth reporting:

- unsafe firmware source validation
- incorrect trust of download URLs or file paths
- privilege escalation risks
- insecure handling of scheduled tasks or admin-authorized commands
- unexpected access to protected user folders

## Disclosure

Please allow time to investigate and fix the issue before public disclosure.
